What HexTTs Is

HexTTs is a local VITS-style text-to-speech research project. It covers the full path from raw LJSpeech data to cleaned phoneme metadata, cached or raw dataloading, model training, checkpointing, inference, and waveform evaluation.

Project Details

HexTTs is a local, end-to-end neural text-to-speech engineering project focused on practical model training, reliability, and explainability rather than API-only demos.

Core Objective

Build and document a reproducible VITS-style TTS workflow that runs on consumer hardware, then track every major architectural and operational change in one timeline.

Technology Stack

  • Python + PyTorch for model/training runtime
  • Librosa + torchaudio for audio processing and alignment
  • HiFi-GAN optional vocoder path for higher quality synthesis
  • Config-driven execution through YAML profiles

Dataset + Inputs

  • Primary dataset: LJSpeech (`data/LJSpeech-1.1`)
  • Prepared metadata: `data/ljspeech_prepared`
  • Real phoneme duration targets: `data/ljspeech_prepared/durations`
  • Validation, preprocessing, and audit scripts included

Pipeline

  1. Validate and preprocess dataset
  2. Generate phoneme metadata and duration targets
  3. Train VITS-style model with checkpoints/logs
  4. Run inference (Griffin-Lim or HiFi-GAN)
  5. Evaluate waveform quality metrics

Key Entrypoints

scripts/prepare_data.py scripts/precompute_features.py scripts/train.py scripts/infer.py scripts/evaluate_tts_output.py scripts/main_flow.py
Current reality check: output quality is improving, but this is still an active research build. The project prioritizes visible engineering decisions, diagnostics, and honest documentation over pretending everything is magically solved.
